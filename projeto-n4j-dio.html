📖Sobre o Projeto

Este projeto foi desenvolvido como desafio do Bootcamp Neo4j e Análise de Dados com Grafos pela DIO. Consiste na criação de um Grafo de Conhecimento que simula um serviço de streaming, modelando as relações complexas entre usuários, filmes, séries, atores, diretores e gêneros.

🎯 Objetivo do Desafio: Demonstrar domínio em modelagem de dados com grafos, criando um mínimo de 10 nós por entidade e estabelecendo relacionamentos significativos entre eles.

✨ Funcionalidades Principais

👥 Gestão de Usuários

Sistema completo de usuários com avaliações de filmes e séries

🎬 Catálogo de Mídia

Filmes e séries organizados por gêneros e relacionamentos

🎭 Elenco e Direção

Mapeamento de atores e diretores com seus papéis

// Usuários assistem e avaliam (:User)-[:RATED {rating: 4.5}]->(:Movie) // Atores atuam em filmes (:Person)-[:ACTED_IN]->(:Movie) // Diretores dirigem filmes (:Person)-[:DIRECTED]->(:Movie) // Filmes têm gêneros (:Movie)-[:HAS_GENRE]->(:Genre)

💻Como Usar

1. Pré-requisitos

Certifique-se de ter instalado:

  • 🖥️ Neo4j Desktop ou Neo4j Aura
  • 💻 Cypher Shell ou Neo4j Browser
  • 📦 Git instalado

2. Clone o Repositório

git clone git@github.com:nataliaberbetviana/neo4j-movie-knowledge-graph.git cd neo4j-movie-knowledge-graph

3. Configure o Neo4j

1. Abra o Neo4j Desktop
2. Crie um novo banco de dados
3. Inicie o banco de dados
4. Abra o Neo4j Browser

4. Execute os Scripts

// 1️⃣ Criar restrições e índices :source scripts/01_create_constraints.cypher // 2️⃣ Criar dados dos CSVs no Neo4J :source scripts/02_create_data.cypher // 3️⃣ Criar relações entre os dados :source scripts/03_create_relations.cypher // 4️⃣ Executar queries de teste :source scripts/04_test_queries.cypher

🔍Consultas de Exemplo

📊 Top Filmes Mais Assistidos

MATCH (u:User)-[w:WATCHED]->(m:Movie) RETURN m.title AS Filme, count(w) AS TotalVisualizações, avg(w.rating) AS AvaliaçãoMédia ORDER BY TotalVisualizações DESC LIMIT 10;

🎭 Colaborações Frequentes

MATCH (d:Person)-[:DIRECTED]->(m:Media)<-[:ACTED_IN]-(a:Person) WHERE d.nome = "Christopher Nolan" WITH a, count(m) AS colaborações WHERE colaborações > 1 RETURN a.nome AS Ator, colaborações ORDER BY colaborações DESC;

🌟 Atores Mais Populares

MATCH (p:Person)-[:ACTED_IN]->(m:Media)<-[r:RATED]-(u:User) WHERE m.tipo = 'Movie' RETURN p.nome AS Ator, count(DISTINCT m) AS Filmes, count(r) AS TotalAvaliacoes, avg(r.rating) AS AvaliaçãoMédia ORDER BY TotalAvaliacoes DESC LIMIT 10;

🎓Aprendizados

Este projeto demonstra habilidades avançadas em modelagem de dados não-relacionais utilizando banco de dados de grafos. A estrutura permite análises complexas de relacionamentos que seriam difíceis ou ineficientes em bancos relacionais tradicionais.

Desenvolvido como parte do Bootcamp Neo4j pela DIO, este projeto representa domínio em análise de dados com grafos, modelagem de relacionamentos complexos e criação de sistemas de recomendação inteligentes.

✨ Destaques do Projeto

  • ✅ Sistema completo de avaliações (ratings de 0 a 5)
  • ✅ Recomendações baseadas em preferências de gênero
  • ✅ Análise de padrões de visualização e colaboração
  • ✅ Identificação de colaborações frequentes entre diretores e atores
  • ✅ Rankings de popularidade e média de avaliações
  • ✅ Queries otimizadas com índices e constraints
